開発陣向け Issue の使い方

このドキュメントでは、Issue をどのように運用していくか説明します。

ラベル

ラベルは、Issue の種類を識別しやすくします。振り分けはやっておきます。
開発を進める優先度としては、Bugs > Other changes > Feature Requests といった感じです。(Feature Discussions は固まるまでは触れないほうがいい気がします。)

  • Bugs
    不具合を表します。
  • Feature Requests
    機能追加のリクエストです。
  • Feature Discussions
    機能についての議論です。機能追加のリクエストであっても、議論の余地があるものについてはこちらに振り分けます。
  • Other changes
    小さな変更点のリクエストです。
  • Questions
    質問です。

Assign

Assign 機能を使って、現在実装にあたっている人を表します。自ら Issue に取り組むときは、必ず自分に Assign をするようにしてください。
